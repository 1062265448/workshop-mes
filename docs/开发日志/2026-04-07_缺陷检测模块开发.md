### 6. Bug 修复与优化 (10:55 - 11:15)

#### 6.1 端口占用问题
**问题**: NestJS watch 模式自动重启导致端口占用

**现象**:
```
Error: listen EADDRINUSE: address already in use :::3001
```

**原因**:
- 代码修改触发自动重新编译
- 旧进程未停止，新进程无法绑定端口

**解决方案**:
- 建议创建自动重启脚本
- 养成 Ctrl+C 停止服务的习惯
- 批量修改代码减少重启次数

**发生次数**: 5 次

---

#### 6.2 API 参数验证问题
**问题**: API 返回 400 错误

**原因**: `ParseIntPipe` 期望字符串参数，前端传数字

**解决方案**:
```typescript
// 移除严格的参数验证
@Query('defectTypeId') defectTypeId?: string,
@Query('page') page?: string,
@Query('limit') limit?: string,

// 在 Service 层手动转换
const dtId = defectTypeId ? parseInt(defectTypeId) : undefined;
const p = page ? parseInt(page) : 1;
const l = limit ? parseInt(limit) : 20;
```

---

#### 6.3 上传字段名不匹配
**问题**: 上传返回 "Unexpected field" 400 错误

**原因**: 前端 el-upload 的 name 属性未设置，与后端 `FileInterceptor('image')` 不匹配

**解决方案**:
```vue
<el-upload
  name="image"
  ...
/>
```

---

#### 6.4 defectTypeId 为数组
**问题**: defectTypeId 被传递为数组 `[1]` 而不是单个值 `1`

**原因**: el-select 绑定问题

**解决方案**:
```typescript
const dtId = Array.isArray(uploadForm.defectTypeId) 
  ? uploadForm.defectTypeId[0] 
  : uploadForm.defectTypeId;
```

---

#### 6.5 图片路径问题
**问题**: 前端请求的是 `http://localhost:5173/uploads/...` 而不是 `http://localhost:3001/uploads/...`

**原因**: Vite 代理未配置 `/uploads/` 路径

**解决方案**:
```typescript
// vite.config.ts
proxy: {
  '/api': {
    target: 'http://localhost:3001',
    changeOrigin: true,
  },
  '/uploads': {
    target: 'http://localhost:3001',
    changeOrigin: true,
  },
}
```

**结果**: ✅ 图片正常显示

```prisma
model DefectType {
  samples     DefectSample[]
  annotations DefectAnnotation[]
}

model DefectSample {
  defectType  DefectType @relation(fields: [defectTypeId], references: [id])
  annotations DefectAnnotation[]
}

model DefectAnnotation {
  sample     DefectSample @relation(fields: [sampleId], references: [id])
  defectType DefectType   @relation(fields: [defectTypeId], references: [id])
}
```
